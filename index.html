formularios HTML

Ideas

los elementos con los que los usuarios pueden interactuar para proporcionar datos se conocen como campos de formulario. A cada campo de formulario se le asigna un nombre en el HTML, que lo identifica para la aplicación de servidor, pero no es visible para el usuario.

Apuntes

Aunque los diversos elementos de un formulario parecen diferentes para el usuario, todos ellos transmiten información a la aplicación de servidor como una serie de pares de nombre y valor, separados por caracteres & . Cada nombre es el nombre de un campo de formulario y el valor son los datos producidos por las acciones del usuario.

Resumen

os formularios son áreas delimitadas por una etiqueta (form), que contiene recuadros de entrada de texto, botones, recuadros de selección y otras características de una interfaz gráfica de usuario. Las aplicaciones web utilizan los formularios para permitir que los usuarios proporcionen los datos que se van a enviar al servidor

formularios CSS

Ideas

display block: siempre comienza en una linea y ocupa todo el ancho disponible

inline: no comienza en una nueva linea y solo ocupa un ancho necesario

inline-block: permite establecer un hancho y una altura en el elemento

none: se usa comunmente con JavaScript para ocultar y mostrar elemento cuando el usuario realiza alguna accion

Apuntes

Gracias a los estilos podemos romper el diseño gris con líneas negras de los formularios y convertirlos en una parte más que se integra perfectamente en nuestro diseño web

color

Ideas

Cuando se crea un sitio web, la eleccion de los colores es una tarea delicada, a pesar de parecer facil. Seleccionar una paleta de colores correcta es importante para comunicar eficasmente el mensaje.

Apuntes

Aunque no hay reglas absolutas, tener en cuenta algunas pautas pueden ayudarnos a encontrar un diseno armonico.

-identifica el publico objetivo de tu pagina web

-no abusar de los colores

-transmitir sensaciones

Resumen

Los colores en un sitio web pueden ser una herramienta muy eficas para lograr que los visitantes sde sientan comodos e interactuen en el sitio.

Ideas

Apuntes

Cada selector tiene su lugar en la jerarquía de especialidad. Hay cuatro categorías que definen el nivel de especificidad de un selector:

- estilos inline

- ID´S

- clases, atributos y pseudo-clases

- elementos y pseudoelementos

Resumen

Solo se aplica cuando el mismo elemento es el objetivo de varias declaraciones. cuando varias declaraciones tienen la misma especificidad, la ultima declaración encontrada en CSS se aplica al elemento.

after y before

Ideas

Si bien el resultado final no esta realmente en el documento, apacerceen la pagina como si lo estuviera. Se debe usar la propiedad de contenido (content) para especificar el contenido para insertar.

Apuntes

El selectoctor BEFORE inserta algo andes del contenido de cada uno de los elementos.

Resumen
